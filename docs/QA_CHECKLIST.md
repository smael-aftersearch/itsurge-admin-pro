# QA Checklist

Use this checklist before publishing a new release.

## Layout

- [ ] Sidebar collapse keeps icons centered.
- [ ] Components submenu opens and closes.
- [ ] Authentication submenu opens and closes.
- [ ] Active submenu item is visible after navigation.
- [ ] Footer stays at the bottom on short pages.
- [ ] Light and dark modes are readable.

## Entity pages

- [ ] Users page has user-specific columns and modal fields.
- [ ] Roles page has role-specific cards and permission matrix toggles.
- [ ] Products page has product-specific columns and modal fields.
- [ ] Orders page has order-specific columns and modal fields.
- [ ] Reports page has report-specific columns and modal fields.
- [ ] Add/Create buttons open modals.
- [ ] View buttons open details modals.
- [ ] Edit buttons open prefilled edit modals.
- [ ] Delete buttons open confirmation modals.

## Apps

- [ ] Calendar event click opens details/edit modal.
- [ ] Add Event opens event modal.
- [ ] To‑Do Add Task opens task modal.
- [ ] To‑Do edit/delete/complete works.
- [ ] Chat contact click switches messages.
- [ ] Chat Call/Profile/Edit opens drawer.

## Auth

- [ ] Login 1/2/3 render correctly.
- [ ] Register 1/2/3 render correctly.
- [ ] Forgot Password 1/2/3 render correctly.
- [ ] 2FA 1/2/3 render correctly without overflow.
- [ ] Persian auth pages are RTL.
- [ ] English auth pages are LTR.

## Visual quality

- [ ] Avatar initials are centered and readable.
- [ ] Borders and shadows are consistent.
- [ ] Font weights are not overly bold.
- [ ] Color palette is visually balanced.
