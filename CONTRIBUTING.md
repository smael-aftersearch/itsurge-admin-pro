# Contributing

Thanks for contributing to ITSurge Admin Pro.

## Development guidelines

- Keep English pages LTR and Persian pages RTL.
- Do not reuse the wrong entity modal across pages. Users, Roles, Products, Orders and Reports must keep their own fields.
- Test Add, View, Edit and Delete actions before submitting changes.
- Keep typography readable and avoid excessive bold weights.
- Use semantic HTML where possible.
- Keep JavaScript lightweight and dependency-free.

## Local review checklist

Before opening a pull request:

1. Open `index.html` and `index-fa.html`.
2. Test sidebar collapse and submenu expand/collapse.
3. Test Add/Edit/View/Delete on Users, Roles, Products, Orders and Reports.
4. Test Calendar event add/edit/delete.
5. Test To‑Do add/edit/delete/complete.
6. Test Chat conversation switching and drawer actions.
7. Test Login/Register/Forgot/2FA variants in English and Persian.
8. Check mobile width around 390px and tablet width around 768px.
