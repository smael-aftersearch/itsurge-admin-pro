## معرفی

**ITSurge Admin Pro** یک قالب حرفه‌ای، واکنش‌گرا و دوزبانه برای ساخت پنل‌های مدیریت مدرن است. این قالب شامل نسخه فارسی راست‌به‌چپ و نسخه انگلیسی چپ‌به‌راست، صفحه‌های کاربردی، کامپوننت‌های قابل استفاده مجدد و رفتارهای تعاملی نمونه با HTML، CSS و JavaScript است.

هدف این قالب این است که یک نقطه شروع تمیز، قابل توسعه و نزدیک به محصول واقعی برای تیم‌هایی فراهم کند که می‌خواهند سریع‌تر یک پنل مدیریت زیبا و قابل ارائه داشته باشند، بدون اینکه از ابتدا همه جزئیات طراحی، چیدمان، فرم‌ها، مودال‌ها و کامپوننت‌ها را بسازند.

این قالب برای داشبوردهای SaaS، پنل‌های CRM و ERP، پنل مدیریت فروشگاه اینترنتی، پورتال‌های داخلی سازمانی، داشبوردهای تحلیل داده و پروژه‌هایی مناسب است که هم رابط فارسی و هم رابط انگلیسی نیاز دارند.

## شروع سریع

برای مشاهده نسخه انگلیسی، فایل `index.html` را در مرورگر باز کنید.

برای مشاهده نسخه فارسی، فایل `index-fa.html` را در مرورگر باز کنید.

برای اجرای لوکال با یک سرور ساده:

```bash
python -m http.server 8080
```

سپس آدرس نسخه انگلیسی را باز کنید:

```text
http://localhost:8080
```

یا آدرس نسخه فارسی را باز کنید:

```text
http://localhost:8080/index-fa.html
```

## سفارشی‌سازی

برای تغییر نام برند، لوگو و favicon، فایل‌های داخل مسیر `assets/img` را بررسی کنید. همچنین می‌توانید عبارت `ITSurge Admin Pro` را در فایل‌های HTML جستجو کرده و با نام محصول خود جایگزین کنید.

متغیرهای اصلی رنگ، سطح‌ها، border، سایه‌ها و تایپوگرافی در فایل‌های داخل مسیر `assets/css` قرار دارند. بهتر است تغییرات ظاهری اصلی را از متغیرهای CSS شروع کنید تا هماهنگی کل قالب حفظ شود.

رفتارهای نمایشی مثل مودال‌ها، پیام‌ها، جدول‌ها، تقویم، وظایف و چت در فایل‌های داخل مسیر `assets/js` قرار دارند. این بخش‌ها ساده نگه داشته شده‌اند تا در زمان اتصال به پروژه واقعی بتوانید آن‌ها را با منطق فریم‌ورک یا API جایگزین کنید.

## اتصال به پروژه واقعی

این قالب مستقل از فریم‌ورک است و می‌تواند با Angular، React، Vue، ASP.NET Core، Laravel، Node.js، REST API یا GraphQL API استفاده شود.

پیشنهاد برای استفاده در پروژه واقعی:

1. بخش‌های تکراری مثل Layout، Sidebar، Topbar، Modal و Table را به کامپوننت تبدیل کنید.
2. داده‌های نمونه را با داده‌های واقعی API جایگزین کنید.
3. اعتبارسنجی فرم‌ها را به منطق اصلی پروژه وصل کنید.
4. مدیریت نقش‌ها و سطح دسترسی را از سمت بک‌اند کنترل کنید.
5. رفتار RTL و LTR را در یک نقطه مرکزی مدیریت کنید.
